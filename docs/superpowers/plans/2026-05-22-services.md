# Self-Hosted Docker — Services Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create Vaultwarden, Forgejo, Jellyfin, and static site stacks. Each is an independent directory. Foundation plan must be deployed first (Traefik, Authelia, Komodo, Backup must be running).

**Architecture:** All stacks follow `_template/` pattern. Only web-facing containers join the `proxy` network. Postgres DBs on internal networks only. Backup labels on all stateful services. OIDC configured post-deploy in Forgejo and Jellyfin UI.

**Tech Stack:** Vaultwarden 1.32.7-alpine, Forgejo 9.0.3, Jellyfin 10.10.7, Postgres 17.10-alpine, nginx 1.27-alpine, sops+age

> **Auth note:** Vaultwarden does not support OIDC. It uses forward auth via Authelia (the `chain-default@file` middleware). Authelia gates the subdomain; inside Vaultwarden users authenticate with their vault master password.

> **DB URL note:** Vaultwarden's `DATABASE_URL` must be a complete connection string — it does not support `_FILE` env vars for this. The URL is assembled in `make up` from decrypted secret files and passed as a process-level env var. It appears in `docker inspect` but the Postgres instance is on an internal network only.

- [ ] **Step 2: Generate Vaultwarden admin token hash**

```bash
# Generate raw admin token
openssl rand -hex 32
# → e.g. a3f9b2c1d4e5...  (store in password manager)

# Hash it for ADMIN_TOKEN env var
docker run --rm vaultwarden/server:1.32.7-alpine \
  /vaultwarden hash --preset owasp
# Enter password when prompted, use the raw token from above
# Output: $argon2id$v=19$m=65536,t=3,p=4$...
# Store hash — you'll put it in .env.sops as VW_ADMIN_TOKEN_HASH
```

- [ ] **Step 3: Create `vaultwarden/Makefile`**

```makefile
STACK_NAME  := vaultwarden
SECRETS_DIR := /dev/shm/$(STACK_NAME)-secrets
SOPS_FILE   := .env.sops
COMPOSE     := docker compose -f compose.yaml

.PHONY: secrets-init secrets-clean up down logs ps encrypt decrypt edit

secrets-init:
	@mkdir -p $(SECRETS_DIR) && chmod 700 $(SECRETS_DIR)
	@sops --decrypt --extract '["DB_PASSWORD"]'          $(SOPS_FILE) > $(SECRETS_DIR)/db_password
	@sops --decrypt --extract '["VW_ADMIN_TOKEN_HASH"]'  $(SOPS_FILE) > $(SECRETS_DIR)/admin_token_hash
	@chmod 600 $(SECRETS_DIR)/*
	@echo "Secrets written to $(SECRETS_DIR) (RAM only)"

secrets-clean:
	@rm -rf $(SECRETS_DIR)
	@echo "Secrets wiped from $(SECRETS_DIR)"

up: secrets-init
	@DOMAIN=$$(sops --decrypt --extract '["DOMAIN"]' $(SOPS_FILE)) \
	 VW_DB_URL="postgresql://vaultwarden:$$(cat $(SECRETS_DIR)/db_password)@postgres:5432/vaultwarden" \
	 VW_ADMIN_TOKEN_HASH="$$(cat $(SECRETS_DIR)/admin_token_hash)" \
	 $(COMPOSE) up -d

down:
	$(COMPOSE) down
	$(MAKE) secrets-clean

logs: ; $(COMPOSE) logs -f
ps:   ; $(COMPOSE) ps

encrypt:
	sops --encrypt .env > $(SOPS_FILE)
	rm -f .env
	@echo "Encrypted. Commit $(SOPS_FILE), keep .env out of git."

decrypt:
	sops --decrypt $(SOPS_FILE) > .env
	@echo "WARNING: .env is plaintext. Delete after use: rm .env"

edit:
	sops $(SOPS_FILE)
```
